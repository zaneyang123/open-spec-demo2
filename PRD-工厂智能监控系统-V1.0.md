## 2. 需求背景与目标

### 2.1 业务背景

随着工厂数字化管理的推进，车间现场管理面临新的挑战。目前车间存在以下管理难点：

1. **手机偷拍问题**：部分员工在工作时间使用手机拍摄现场工作环境、产品信息等敏感内容，存在商业机密泄露风险
2. **员工工作效率问题**：部分员工在工作时间存在偷懒行为，影响生产效率和产品质量
3. **管理成本高**：传统人工巡检方式效率低、覆盖范围有限，难以实现24小时全天候监管

为了规范车间管理，提升现场管理效率，需要建设一套智能化的视觉监控系统，实现对员工违规行为的自动检测和告警。

### 2.2 需求痛点

| 痛点 | 具体表现 | 影响 |
|------|----------|------|
| 痛点1 | 手机偷拍现场工作环境和产品 | 商业机密泄露风险，给企业造成不可估量的损失 |
| 痛点2 | 员工偷懒 | 生产效率下降，产品质量无法保证，人工管理成本高 |

### 2.3 需求目标

| 目标类型 | 具体指标 |
|----------|----------|
| 检测正确率 | ≥ 99% |
| 系统可用性 | 24小时运行 |
| 硬件成本 | 单个摄像头 < 500元 |
| 整体预算 | 整套系统 ≤ 20万元 |
| 摄像头数量 | 约50个点位 |

### 2.4 价值说明

| 价值维度 | 说明 |
|----------|------|
| 对用户 | 减少人工巡检工作量，实现自动化监管；及时发现违规行为，提升管理效率 |
| 对业务 | 规范车间管理，保护商业机密；提升员工工作效率，保障产品质量 |
| 对技术团队 | 积累视觉AI监控技术经验，为后续智能制造项目奠定基础 |

## 3. 目标用户与使用场景

### 3.1 目标用户

| 角色 | 人数 | 主要职责 |
|------|------|----------|
| 车间管理人员 | 若干 | 负责车间日常管理，监控员工行为，处理违规告警 |
| 车间主管/班组长 | 若干 | 接收违规告警，处理现场问题，汇总分析数据 |

### 3.2 使用场景

| 场景 | 时间 | 地点 | 行为 | 期望结果 |
|------|------|------|------|----------|
| 实时监控 | 工作时间 | 车间监控区域 | 通过监控画面查看现场情况 | 实时掌握现场状态，发现异常 |
| 违规告警 | 实时 | 监控区域 | 员工出现玩手机行为时 | 触发声光告警，推送消息给主管 |
| 视频追溯 | 事后 | 监控室/办公室 | 查看历史违规视频 | 确认违规事实，进行处理 |
| 数据统计 | 定期 | 办公室 | 查看违规统计数据 | 分析违规趋势，优化管理 |

## 4. 核心功能需求

### 4.1 功能模块总览

| 模块 | 功能描述 | 优先级 |
|------|----------|--------|
| F1 | 实时检测 | P0 |
| F2 | 声光告警 | P0 |
| F3 | 视频记录 | P0 |
| F4 | 告警推送 | P0 |
| F5 | 数据统计 | P1 |
| F6 | 报表导出 | P1 |
| F7 | 权限管理 | P1 |
| F8 | 设备管理 | P2 |

### 4.2 详细功能描述

#### F1 实时检测

| 项目 | 内容 |
|------|------|
| 功能名称 | 实时检测 |
| 功能描述 | 通过视觉AI算法实时分析摄像头画面，检测员工是否存在玩手机行为 |
| 用户交互流程 | 1. 系统自动采集摄像头视频流<br>2. AI算法实时分析画面<br>3. 检测到违规行为时标记目标<br>4. 在监控画面上显示告警框选 |
| 输入 | 摄像头视频流 |
| 输出 | 违规检测结果（包含时间、位置、置信度） |
| 边界条件 | - 光线过暗时可能影响检测准确率<br>- 多人同时出现在画面时的处理<br>- 摄像头角度和覆盖范围 |
| 非功能性需求 | - 检测响应时间 < 2秒<br>- 准确率 ≥ 99% |

#### F2 声光告警

| 项目 | 内容 |
|------|------|
| 功能名称 | 声光告警 |
| 功能描述 | 检测到违规行为时，通过声光方式发出告警，提醒现场人员 |
| 用户交互流程 | 1. 收到违规检测结果<br>2. 触发声光告警设备<br>3. 播放告警声音/闪烁警灯<br>4. 持续一定时间后自动停止 |
| 输入 | 违规检测结果 |
| 输出 | 声光信号输出 |
| 边界条件 | - 告警声音音量可调<br> - 告警持续时间可配置<br>- 避免误报导致频繁告警 |
| 非功能性需求 | - 告警触发延迟 < 1秒<br>- 声音可覆盖车间范围 |

#### F3 视频记录

| 项目 | 内容 |
|------|------|
| 功能名称 | 视频记录 |
| 功能描述 | 对监控区域进行视频录制，并保存违规片段供事后追溯 |
| 用户交互流程 | 1. 系统持续录制监控视频<br>2. 检测到违规时自动标记时间点<br>3. 保存违规前后视频片段<br>4. 支持按时间/摄像头回放 |
| 输入 | 摄像头视频流 |
| 输出 | 视频文件（存储在本地或云端） |
| 边界条件 | - 存储空间限制<br>- 视频清晰度要求<br>- 存储周期要求 |
| 非功能性需求 | - 视频分辨率 ≥ 1080P<br>- 存储周期：待确认<br>- 支持快速检索和回放 |

#### F4 告警推送

| 项目 | 内容 |
|------|------|
| 功能名称 | 告警推送 |
| 功能描述 | 将违规告警信息推送给相关管理人员 |
| 用户交互流程 | 1. 检测到违规行为<br>2. 生成告警消息（含时间、位置、图片/视频片段）<br>3. 推送给车间主管/班组长<br>4. 管理人员确认处理 |
| 输入 | 违规检测结果 |
| 输出 | 推送消息（APP/短信/即时通讯） |
| 边界条件 | - 推送失败的重试机制<br>- 消息堆积处理<br>- 不同角色的推送范围 |
| 非功能性需求 | - 推送延迟 < 5秒<br>- 支持多种推送方式 |

#### F5 数据统计

| 项目 | 内容 |
|------|------|
| 功能名称 | 数据统计 |
| 功能描述 | 对违规数据进行统计分析，生成统计报表 |
| 用户交互流程 | 1. 用户进入统计页面<br>2. 选择统计维度（时间/区域/人员）<br>3. 查看统计图表和数据<br>4. 支持下钻查看明细 |
| 输入 | 违规记录数据 |
| 输出 | 统计数据（图表+数值） |
| 边界条件 | - 统计时间范围选择<br>- 多维度交叉分析 |
| 非功能性需求 | - 统计响应时间 < 3秒 |

#### F6 报表导出

| 项目 | 内容 |
|------|------|
| 功能名称 | 报表导出 |
| 功能描述 | 将统计数据导出为指定格式的报表文件 |
| 用户交互流程 | 1. 用户选择报表类型和时间范围<br>2. 点击导出按钮<br>3. 生成报表文件<br>4. 下载到本地 |
| 输入 | 统计筛选条件 |
| 输出 | Excel/PDF报表文件 |
| 边界条件 | - 导出数据量限制<br>- 文件格式兼容性 |
| 非功能性需求 | - 支持Excel和PDF格式 |

#### F7 权限管理

| 项目 | 内容 |
|------|------|
| 功能名称 | 权限管理 |
| 功能描述 | 对系统用户进行权限分级管理，控制不同角色的访问范围 |
| 用户交互流程 | 1. 管理员创建用户账号<br>2. 分配角色和权限<br>3. 用户登录系统<br>4. 根据权限展示功能菜单 |
| 输入 | 用户信息、角色信息 |
| 输出 | 权限控制结果 |
| 边界条件 | - 角色类型定义<br>- 权限粒度控制 |
| 非功能性需求 | - 支持分级权限（管理员/主管/查看员） |

#### F8 设备管理

| 项目 | 内容 |
|------|------|
| 功能名称 | 设备管理 |
| 功能描述 | 对摄像头和告警设备进行集中管理和监控 |
| 用户交互流程 | 1. 查看设备在线状态<br>2. 查看设备基本信息<br>3. 远程重启/配置设备<br>4. 设备异常告警 |
| 输入 | 设备状态数据 |
| 输出 | 设备管理界面 |
| 边界条件 | - 设备类型多样性<br>- 网络连接状态 |
| 非功能性需求 | - 设备状态实时刷新 |
